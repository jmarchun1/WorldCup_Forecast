# fix: v4.1 Dead-Rubber, Tilt Context, Version Tracking

## Problem Frame

Three accuracy gaps remain after v4:

1. **Dead-rubber effect unmodelled** — When both teams in a group-stage match-3 have already qualified (or been eliminated) before kickoff, squads rotate heavily and incentives shift. Research shows draw probability rises ~+8–12pp and goals/match drops ~0.3–0.5 vs normal games. The LLMs are not aware of the pre-match qualification state unless it is explicitly injected; they default to treating every match as fully competitive.

2. **Team attacking/defensive style not quantified** — Nate Silver's PELE model includes a "Tilt" rating measuring whether a team's style pushes matches toward high-scoring or low-scoring outcomes (e.g., Germany/Norway = high-tilt attacking; Colombia/Senegal = negative-tilt defensive). This signal is orthogonal to Elo strength and directly predicts goal totals. It is currently absent from the prompt.

3. **No cross-version performance comparison** — The accuracy tab shows per-model stats for the current archive only. We cannot see whether v4 is actually better than v3 by comparing P/G, exact-score rate, or Brier score across archive versions side-by-side in the dashboard.

**Scope:** `forecast.py` (prompt injection + match-day detection), `push_to_pages.py` + `docs/_template.html` (cross-version accuracy comparison table).

- Match-day 3 dead-rubber flag injected for games where both teams have pre-match qualification state computable from results so far
- PELE Tilt context for ~20 key teams injected into relevant matches
- Accuracy tab shows a cross-version comparison table: P/G, exact%, draw_pred%, Brier per archive version

## Key Technical Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Dead-rubber detection | Read `results/scored/` consensus files to derive group standings; compute if team is qualified/eliminated before match-3 | Avoids external dependency; scored data already exists |
| Tilt encoding | Hard-coded `WC2026_TILT` constant with ~20 teams, injected into prompt when home or away team is present | Stable for group stage; update manually for v5 if needed |
| Cross-version comparison | New `_build_cross_version_stats()` in `push_to_pages.py`; reads each archive's scored files | Archives contain scored records; retrospective scoring is possible |
| Qualification state prompt text | Injected as a new `=== MATCH CONTEXT OVERRIDE ===` block when dead-rubber detected | Distinct from existing MATCH CONTEXT so models see it as a hard signal |

## Implementation Units

### U1. Dead-rubber detection and prompt injection

**Goal:** Compute pre-match qualification state for MD3 group-stage games; inject dead-rubber warning into prompt when both teams have nothing to play for (both qualified, or both eliminated).

**Files:**
- `forecast.py` — new `_get_dead_rubber_context(match_data)` function; called in `build_prompt()`

**Approach:**

1. New `_compute_group_standings(group_id)` function:
   - Load `results/scored/` CONSENSUS files where `match_id` matches the group's teams
   - Compute points table (W=3, D=1, L=0) from actual results
   - Return dict: `{team: {pts, gd, gf, played}}`

2. New `_get_dead_rubber_context(match_data)` function:
   - Identify match-day from fixture order (already computable — see fixture analysis)
   - If not MD3: return `None`
   - Compute group standings from prior results
   - Determine qualification state for home and away teams:
     - **Clinched top 2**: pts_after_2_games ≥ 6, or mathematically guaranteed
     - **Eliminated**: pts_after_2_games = 0 and GD so negative that 3 pts won't rescue
     - **Still competing**: everything else
   - Return a context string if both clinched OR both eliminated

3. Inject in `build_prompt()` after `=== MATCH CONTEXT ===`:
   ```
   === MATCH-DAY 3 INCENTIVE CONTEXT ===
   [HOME] has already [qualified / been eliminated]. [AWAY] has already [qualified / been eliminated].
   Research shows that when both teams have nothing to play for, draw probability rises ~10pp above
   the bookmaker implied rate, goals/match drops ~0.4 vs earlier rounds, and squads are typically
   rotated 4-6 players from the starting XI. Weight your prediction accordingly.
   ```

**Simplification:** For now use a rule-based heuristic on points after MD1+MD2:
- Clinched: 6 pts (two wins) — mathematically guaranteed top-2 in 4-team group if other game hasn't both won
- Eliminated: 0 pts with GD ≤ −3 after 2 games (cannot realistically reach 3 pts even with a win)
- Defer exact standing calculation to v5; inject context only for clear-cut cases

**Verification:** Check Algeria vs Austria (MD3, Jun 27) — Algeria has 0 pts from 3-0 loss and Jordan loss; Austria has 3 pts. Only Algeria is likely eliminated; inject one-sided context.

### U2. PELE Tilt injection into prompt

**Goal:** Inject attacking/defensive style context for teams where Tilt is known, to help models calibrate goal-total predictions.

**Files:**
- `forecast.py` — new `WC2026_TILT` constant; inject in `build_prompt()` under `=== HISTORICAL BASE RATES ===`

**Approach:**

Add constant:
```python
WC2026_TILT = {
    # Positive tilt = attacking style, pushes toward higher-scoring matches
    "Germany": "+high", "Norway": "+high", "Netherlands": "+high",
    "England": "+high", "France": "+mid", "Brazil": "+mid",
    "Argentina": "+mid", "Spain": "+mid", "USA": "+mid",
    "Sweden": "+mid", "Portugal": "+mid",
    # Negative tilt = defensive style, pushes toward lower scoring / draws
    "Colombia": "-low", "Senegal": "-low", "Morocco": "-low",
    "Saudi Arabia": "-low", "Iran": "-low", "Egypt": "-low",
    "Algeria": "-low", "Uruguay": "-low", "Japan": "-low",
}
```

In `build_prompt()`, after HISTORICAL BASE RATES section:
```
=== TEAM STYLE (PELE TILT) ===
{Home}: {tilt description} — {implication}
{Away}: {tilt description} — {implication}
```
Only inject when at least one team has a known Tilt. Skip section if both neutral/unknown.

Tilt descriptions:
- `+high`: "attacking — games involving this team tend toward 3+ goals total"
- `+mid`: "moderately attacking — games tend toward 2.5–3 goals total"  
- `-low`: "defensive — games tend toward draws and under 2.5 goals total"

**Verification:** After re-run, check Norway vs France (Group I MD3) — Norway is +high tilt, should see higher draw_prob and higher goal predictions vs v3.

### U3. Cross-version accuracy comparison in dashboard

**Goal:** Show a side-by-side comparison of P/G, exact score rate, draw prediction rate, and Brier score across all archived forecast versions, so we can objectively measure whether each version improves accuracy.

**Files:**
- `push_to_pages.py` — new `_build_cross_version_accuracy()` function
- `docs/_template.html` — new table in accuracy tab

**Approach:**

**`push_to_pages.py`:**
```python
def _build_cross_version_accuracy() -> list:
    """For each archive version that has scored records, compute aggregate accuracy metrics."""
```
- For each archive stamp dir in `results/forecasts/archive/`:
  - Load the `*_CONSENSUS.json` files from that archive dir
  - Cross-reference with `results/scored/` to get actual results
  - Compute: total_pts, matches_scored, pts_per_game, exact_pct, draw_pred_pct (% of games where draw_prob was highest), brier_mean
- Return list of `{version_id, pts_per_game, exact_pct, draw_pred_pct, brier_mean, matches}`
- Inject as `__CROSS_VERSION_JSON__` placeholder

**`_template.html`:**
- Add `const CROSS_VERSION = __CROSS_VERSION_JSON__;` constant
- In accuracy tab IIFE, after version history table:
  ```
  === Version Accuracy Comparison ===
  | Version | Matches | P/G | Exact% | Draw Pred% | Brier |
  ```
- Highlight best value in each column (green background)
- Note: only versions with ≥10 scored matches shown (skip fresh archives with no scored data)

**Verification:** After deploy, accuracy tab shows ≥2 rows in cross-version table (v3 and v4 archives both have scored data).

## Deferred Implementation Notes

- Exact points-table qualification math (tiebreakers, goal difference, head-to-head) is complex; the heuristic (6 pts = clinched, 0 pts + bad GD = eliminated) covers ~80% of dead-rubber cases cleanly.
- Live PELE ratings API would replace the hard-coded tilt constant in v5 if Nate Silver publishes data.
- Cross-version scoring requires that archive consensus files were produced with the same scoring schema as current `score.py` — verify before assuming retroactive scoring is valid.
